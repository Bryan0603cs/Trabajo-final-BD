Ayudas y manual de usuario

1. Inicio de sesión

Para acceder a la aplicación, el usuario debe ingresar su nombre de usuario y contraseña en la pantalla de login. Si las credenciales son correctas, el sistema registra la entrada en la tabla BITACORA_SESION y habilita el menú principal.

  • Administrador: Puede usar todos los módulos, crear usuarios y consultar la bitácora.
  • Paramétrico: Registra ventas, administra clientes, productos y demás entidades, pero no usuarios.
  • Esporádico: Solo puede realizar consultas y generar reportes.

2. Módulo de Entidades

En este módulo se administran las tablas maestras del sistema: usuarios, clientes, categorías, proveedores, productos, compras y demás entidades. El administrador define la estructura de los datos y el paramétrico mantiene la información actualizada.

Desde aquí se garantiza la integridad de la base de datos, evitando registros huérfanos y manteniendo consistencia en las claves foráneas.

3. Transacciones (Ventas y Créditos)

El módulo de transacciones permite registrar ventas de contado y ventas a crédito. Cada venta genera un registro en VENTAS y sus productos asociados se almacenan en DETALLE_VENTAS.

Cuando la venta es a crédito, automáticamente se crea un registro en CREDITOS, con el saldo total y la fecha de vencimiento. Cada abono del cliente se almacena en la tabla PAGOS.

4. Reportes y Consultas

Este módulo concentra los reportes obligatorios del proyecto (factura de venta, totales mensuales, IVA por trimestre, comparación crédito/contado, inventario por categoría y clientes con créditos en mora), todos en formato PDF.

Además, se plantean consultas multitabla adicionales (por ejemplo, productos más vendidos, clientes con mayor compra acumulada, ventas por categoría, etc.) que ayudan a analizar el comportamiento del negocio.

5. Utilidades

En la opción de utilidades se incluyen herramientas de apoyo como una calculadora rápida para valores de venta y un calendario simple para visualizar fechas importantes (cortes de mes, vencimiento de créditos, etc.).

6. Bitácora de sesiones

Cada vez que un usuario inicia o cierra sesión se registra un evento en la tabla BITACORA_SESION con la fecha y hora correspondientes. Este módulo solo está disponible para el administrador y permite hacer seguimiento de la actividad del sistema.
