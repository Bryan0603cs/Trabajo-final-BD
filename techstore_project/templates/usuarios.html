Descripción general

El módulo de usuarios controla quién puede acceder al sistema y qué operaciones puede realizar. La base de datos utiliza dos tablas principales: USUARIOS y ROLES.

Tabla ROLES

Define los perfiles de acceso al sistema.

Columna Tipo Descripción
ID_ROLNUMBER (PK)Identificador del rol
NOMBRE_ROLVARCHAR2(50)Nombre del rol (ADMIN, PARAMETRICO, ESPORADICO)
DESCRIPCIONVARCHAR2(200)Descripción del nivel de acceso

Tabla USUARIOS

Almacena las credenciales y el rol de cada usuario del sistema.

Columna Tipo Descripción
ID_USUARIONUMBER (PK)Identificador del usuario
NOMBREVARCHAR2(100)Nombre del usuario
APELLIDOVARCHAR2(100)Apellido del usuario
USERNAMEVARCHAR2(50)Nombre de usuario (login)
PASSWORD_HASHVARCHAR2(200)Contraseña encriptada (SHA-256)
ESTADOVARCHAR2(20)ACTIVO / INACTIVO
FECHA_CREACIONDATEFecha de creación del usuario
ID_ROLNUMBER (FK)Referencia al rol asignado

Niveles de acceso

  • ADMIN: Acceso completo (usuarios, entidades, reportes, bitácora).
  • PARAMÉTRICO: Mantiene maestros (clientes, productos, etc.) y registra ventas.
  • ESPORÁDICO: Acceso solo a consultas y reportes.
